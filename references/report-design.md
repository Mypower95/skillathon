# Report Design

리포트는 매장 사장님이 모바일에서 바로 보는 상황을 기준으로 만든다.

## Sections

1. 헤더: 매장명, 기간, 리포트 목적
2. 핵심 지표: 총 주문 금액, 주문 세션 수, 평균 주문 금액, 추가 주문율
3. 인사이트 카드: 추가 주문, 인기 메뉴, 피크 시간대
4. 인기 메뉴 TOP 5
5. 시간대별 주문 패턴
6. 발송 CTA: 담당자가 후속 설명을 드릴 수 있다는 문구

## Tone

- 쉬운 한국어를 사용한다.
- 숫자를 먼저 보여주고, 해석은 짧게 붙인다.
- 과장된 성과 표현은 피한다.
- "매출이 올랐다"처럼 CSV만으로 단정할 수 없는 표현은 쓰지 않는다.

## Visual Rules

- 모바일 폭 390px 안에서도 읽기 쉬운 단일 컬럼 구조를 기본으로 한다.
- 핵심 지표는 카드 형태로 배치한다.
- 색상은 티오더 느낌을 살리되 제출용이므로 과도한 브랜드 세부 규칙은 넣지 않는다.
- 외부 차트 라이브러리 없이 HTML/CSS만으로 재현 가능하게 만든다.
