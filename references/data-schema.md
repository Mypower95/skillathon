# Data Schema

제출용 CSV는 실제 CSV 구조를 유지하되 보안 컬럼을 더미 값으로 바꾼 mock data만 사용한다.

## Required Columns

| Column | Description | Example |
|---|---|---|
| `주문시간` | 주문 시각 | `2026-04-21 18:13:47` |
| `주문ID` | 더미 주문 식별자 | `ORDER-267446` |
| `매장코드` | 더미 매장 코드 | `STORE-459717` |
| `상호지점명` | 제출용 더미 매장명 | `코덱스커뮤니티(최고점)` |
| `태블릿 번호` | 더미 태블릿 번호 | `TABLET-845735` |
| `주문키` | 더미 주문 키 | `KEY-904592` |
| `주문 유형` | 주문/결제 유형 | `후결제` |
| `장바구니` | 메뉴명, 메뉴코드, 수량, 금액이 포함된 주문 문자열 | `[우동(순한맛) (A0013) (1개) (9000원)]` |
| `주문 장바구니 전체 금액` | 해당 주문 행의 총액 | `50500` |
| `최종 포스 연동 여부` | POS 연동 결과 | `성공` |
| `포스 응답 소요 시간` | POS 응답 소요 시간(초) | `1` |

## Derived Metrics

- 총 주문 금액: `quantity * unit_price` 합계
- 주문 세션 수: 고유 `주문ID` 수
- 평균 주문 금액: 총 주문 금액 / 주문 세션 수
- 추가 주문율: 같은 `주문ID`에 2개 이상 메뉴가 있는 세션 비율
- 인기 메뉴: 수량 기준 상위 메뉴
- 매출 기여 메뉴: 매출 기준 상위 메뉴
- 피크 시간대: 주문 수가 가장 많은 시간

## Privacy Rule

제출용 CSV에는 고객명, 전화번호, 이메일, 주소, 결제 승인번호, 내부 매장 코드, 실제 주문 ID, 실제 태블릿 번호, 실제 주문키, 실제 IP를 넣지 않는다.

## Anonymized Columns

`scripts/anonymize_orders.py`는 아래 컬럼을 더미 값으로 치환한다.

- `주문ID` → `ORDER-000000`
- `매장코드` → `STORE-000000`
- `태블릿 번호` → `TABLET-000000`
- `주문키` → `KEY-000000`
- `ip` → `IP-000000`
